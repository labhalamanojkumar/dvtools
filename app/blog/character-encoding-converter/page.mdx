## What is Character Encoding?

At its core, a computer only understands numbers. Character encoding is a rulebook that maps characters (like `A`, `‚Ç¨`, or `üëç`) to specific numerical values.

-   **ASCII:** The original standard, it uses 7 bits to represent 128 characters, covering English letters, numbers, and basic punctuation. It's simple but extremely limited.
-   **ISO-8859-1 (Latin-1):** An 8-bit encoding that extends ASCII to include characters for Western European languages (like `√±`, `√∂`, `√ß`).
-   **Windows-1252:** A Microsoft variant of ISO-8859-1 that replaces some control characters with useful symbols like the Euro sign (`‚Ç¨`) and smart quotes (`‚Äú ‚Äù`).
-   **UTF-8:** The dominant encoding of the web. It's a variable-width encoding that can represent every character in the Unicode standard. It's backward-compatible with ASCII but can also handle virtually any language, symbol, or emoji in the world.

The problem arises when a system doesn't know which rulebook to use.
