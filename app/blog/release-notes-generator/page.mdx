## The Tedious Task of Writing Release Notes

You've just shipped a new version of your software. The code is deployed, and everything is working. But now comes the final, often-dreaded task: writing the release notes. You have to sift through Git logs, check Jira tickets, and manually compile a list of new features, bug fixes, and improvements. It's a time-consuming, error-prone process that developers and product managers alike would rather avoid.

What if you could generate comprehensive, well-formatted release notes in seconds? Our **Release Notes Generator** does exactly that. It connects directly to your development workflow, turning raw data into polished, human-readable updates.

## What is a Release Notes Generator?

A Release Notes Generator is a tool that automates the creation of changelogs and release notes. It analyzes your Git repository's history and integrates with project management systems (like Jira or GitHub Issues) to gather all the necessary information. It then categorizes changes, formats the output, and produces a clean document ready for your customers, stakeholders, or internal teams.

## A Common Use Case: A Weekly Release Cycle

Your team follows a continuous delivery model, deploying to production every Friday. Manually writing release notes for each deployment is becoming a major bottleneck.

1.  **Configure the Generator:** You connect the Release Notes Generator to your GitHub repository and your Jira project. You create a simple template that groups changes into "New Features," "Improvements," and "Bug Fixes."
2.  **Run the Generator:** Every Friday, just before the release, you run the generator with a single command, specifying the last release tag and the current `main` branch.
    *   `releasenotes-gen --from v1.2.0 --to main`
3.  **Automated Compilation:**
    *   The tool scans all commits since `v1.2.0`.
    *   For each commit, it extracts the message. If a commit message includes a Jira ticket ID (e.g., `feat: Add user profile page [PROJ-123]`), it fetches the ticket details from Jira.
    *   It categorizes each item based on the commit prefix (`feat`, `fix`, etc.).
4.  **Instant Output:** Within seconds, the tool generates a perfectly formatted Markdown file:

    ```markdown
    # Release v1.3.0

    ## üöÄ New Features
    - **PROJ-123:** Add a new user profile page to manage account settings.
    - **PROJ-125:** Users can now upload a custom avatar.

    ## üêõ Bug Fixes
    - **PROJ-128:** Fixed a bug where the login button was not visible on mobile devices.
    ```

5.  **Result:** The release notes are ready to be published to your blog, sent in an email, or posted on Slack. The entire process took less than a minute, freeing up your team to focus on what matters most: building great software.
